Files with "product" in their name were calculated with the 'product' flag set to True
This means that the product of the affinity was taken between the first pion's affinity and second pion's affinity - the affinity for each pion was calculated using the calculator (TF model) and inputting the event x, Q2 and the z, pT associated with each pion

Files with "dihadron" in their name were calculated with the 'product' flag set false - these are calculated using the dihadron's z, pT

Files with "ONEPION" in their name were calculated with the product flag on, but the code was adjusted so that only p1's Affinity was used, and no product was calculated
